### How It Works

The community scanner executes a 4-phase search strategy:

1. **Brand Search** — paginated search for "Cursor" across Reddit, HN, Dev.to, Bilibili, V2EX
2. **Category Search** — searches for "devtools" to capture ecosystem discussions
3. **Combo Queries** — `"Cursor review"`, `"Cursor alternative"`, `"best devtools tools"`
4. **Subreddit-Targeted** — searches within `r/programming`, `r/webdev`, `r/devops`, `r/coding`, etc.

All results are deduplicated, scored (text relevance + recency decay + engagement velocity), and stored for historical tracking.

## Reproducing This Example

```bash
# 1. Start OpenCMO
opencmo-web

# 2. Create a monitor for cursor.com
curl -X POST http://localhost:8080/api/v1/monitors \
  -H "Content-Type: application/json" \
  -d '{"url": "https://www.cursor.com", "brand": "Cursor", "category": "devtools", "job_type": "full"}'

# 3. Or use the web UI at http://localhost:8080/app/
# Add "cursor.com" in the monitor form and click "Start Monitoring"
```
